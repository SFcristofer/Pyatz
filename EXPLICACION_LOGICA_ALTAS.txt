==========================================================================
   ARQUITECTURA DE ARCHIVOS PROFESIONAL: BOVEDA 360 PYATZ
==========================================================================

Este documento detalla el funcionamiento del Repositorio Maestro y la 
Boveda Confidencial basada en Bibliotecas de Salesforce.

1. SEGMENTACION DE ALMACENAMIENTO (Seguridad Total)
----------------------------------------------------
El sistema diferencia automaticamente el destino de cada archivo 
segun su categoria:

A) ALTA CLIENTE (Lista 5):
   - Destino: Se vinculan a la CUENTA DEL CLIENTE y a la OPORTUNIDAD.
   - Comparticion: Solo visible para proyectos del MISMO cliente.
   - Privacidad: El Cliente B nunca podra ver documentos del Cliente A.

B) ALTA PYATZ (Lista 6):
   - Destino: BIBLIOTECA "Repositorio Maestro Pyatz".
   - Comparticion: GLOBAL. Sirve para todas las oportunidades de la Org.
   - Eficiencia: Se sube una vez (ej. el REPSE) y se marca como cargado
     en todos los expedientes 360 de la empresa automaticamente.

2. MECANICA TECNICA DE VINCULACION (Tags)
------------------------------------------
El sistema utiliza el TAG (ej. [CL_ACTA] o [PY_REPSE]) como ADN:
- El Agente no necesita renombrar el archivo manualmente.
- El servidor (Apex) detecta el tag y aplica las reglas de Salesforce 
  para Bibliotecas (ShareType = 'I') o Cuentas (ShareType = 'V').
- La busqueda es "Case-Insensitive" para evitar duplicados.

3. CONFIDENCIALIDAD "CIEGA"
----------------------------
Una vez que el archivo es procesado por el sistema:
1. El Agente pierde el acceso de previsualizacion.
2. El boton de carga se bloquea.
3. Aparece la etiqueta "CONFIDENCIAL".
4. Solo el Administrador o Auditor puede ver el archivo real desde
   la seccion estandar de "Notas y Adjuntos" o desde la Biblioteca.

4. MANTENIMIENTO ESCALABLE (Zero Code)
---------------------------------------
Para añadir nuevos documentos, el Administrador solo debe:
1. Ir a Metadatos Personalizados -> Documento Expediente.
2. Crear un registro con el Tag y Categoria deseada.
El Expediente 360 se actualizara en tiempo real para todos los usuarios.
